ftp及lftp

  • 文件传输协议（File Transfer Protocol，FTP），基于该协议FTP客户端与服务端可以实现共享文件、上传文件、下载、删除文件。FTP服务器端可以同时提供给多人共享使用。
  • FTP服务是Client/Server（简称C/S）模式，基于FTP协议实现FTP文件对外共享及传输的软件称之为FTP服务器源端，客户端程序基于FTP协议，则称之为FTP客户端，FTP客户端可以向FTP服务器上传、下载文件。

课堂笔记

ftp 文件传输协议
http 超文本传输协议

vsftpd 服务端
配置文件：/etc/vsftpd/vsftpd.conf
数据默认存放位置：/var/ftp/ # 这里有要求，看下面
第一：不要将/var/ftp这个目录本身设置其他权限
第二：想要实现上传或创建目录，需要使这个目录的属主属组为ftp
第三：注意关闭防火墙和selinux

1、匿名操作

  • 默认情况下，只能下载

2、匿名完成上传下载和创建文件目录
vim /etc/vsftpd/vsftpd.conf # 修改以下部分，然后重启服务
anonymous_enable=YES
anon_umask=022
anon_upload_enable=YES
anon_mkdir_write_enable=YES
anon_other_write_enable=YES
systemctl restart vsftpd

3、使用账户密码登录

  • 创建用户名，密码
    useradd xa2401
    echo “密码” | passwd –stdin xa2401

  • 修改配置文件，支持密码登录，先注释掉所有的匿名操作

    anonymous_enable=YES

​ anon_umask=022

​ anon_upload_enable=YES

​ anon_mkdir_write_enable=YES

​ anon_other_write_enable=YES

  • 新增或修改
    chroot_list_enable=YES
    chroot_list_file=/etc/vsftpd/chroot_list
    allow_writeable_chroot=YES
    chroot_local_user=YES
    local_max_rate=0

  • 添加用户
    vim /etc/vsftpd/chroot_list # 一行一个用户
    xa2401
    systemctl restart vsftpd
